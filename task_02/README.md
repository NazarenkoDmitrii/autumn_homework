# Районы, кварталы, жилые массивы

Фрэнку снова прилетел заказ. В этот раз город устроен странно: все дороги односторонние и соединяют офисы.  
Множество офисов образует **квартал**, если из любого офиса этого множества можно добраться до любого другого (по направлению дорог), и это множество нельзя расширить, сохранив это свойство.  
Иными словами, квартал — это **сильносвязная компонента** ориентированного графа.

Нужно найти **минимальное количество новых односторонних дорог**, которые надо построить, чтобы **весь город стал одним кварталом** (то есть граф стал **сильно связным**).

Нумерация вершин ведётся с **0**. Петли допустимы. Граф может быть несвязным.

## Формат ввода

Первая строка содержит два целых числа `n` и `m`  
(1 ≤ n ≤ 10^5, 0 ≤ m ≤ 10^5) — количество вершин и рёбер ориентированного графа.

Следующие `m` строк содержат по два целых числа `a` и `b` — начало и конец ребра  
(0 ≤ a, b < n) — ориентированное ребро `a -> b`.

## Формат вывода

Выведите одно число — минимальное количество рёбер, которые нужно добавить, чтобы граф стал сильно связным.

## Примеры

### Пример 1
Ввод:
2 1
0 1

Вывод:
1

### Пример 2
Ввод:
4 3
0 1
2 1
1 3

Вывод:
2

### Пример 3
Ввод:
5 4
0 1
2 1
1 3
1 4

Вывод:
2
